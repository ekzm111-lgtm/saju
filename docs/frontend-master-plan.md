# 사주명인 프론트페이지 마스터 기획안

## 1. 프로젝트 목표

사주명인 프론트페이지는 단순 소개 페이지가 아니라, 사용자가 서비스의 신뢰감과 몰입감을 동시에 느끼고 결제까지 자연스럽게 이동하도록 설계하는 것이 핵심이다.

핵심 목표는 아래 4가지다.

- 첫 화면에서 "프리미엄 AI 사주 서비스"라는 인상을 강하게 전달한다.
- 정적인 랜딩이 아니라, 부드럽고 인상적인 움직임으로 몰입감을 준다.
- 모바일에서도 불편 없이 탐색하고 결제까지 이어지게 만든다.
- 후기, 상품, 결과 예시를 통해 결제 전 불안감을 줄인다.

브랜드 톤은 아래 방향을 기준으로 한다.

- 신비롭지만 과장되지 않은 분위기
- 전통적인 사주 감성과 현대적인 SaaS 감성의 결합
- 고급스럽고 차분하지만, 화면은 살아있는 느낌

## 2. 사용자 여정

메인 유입 사용자의 기본 여정은 아래 순서를 따른다.

1. 광고, 검색, SNS, 지인 공유로 랜딩 진입
2. 첫 화면에서 서비스 정체성과 매력을 빠르게 파악
3. 어떤 상품이 있는지 확인
4. 후기와 결과 예시를 보고 신뢰 형성
5. 서비스 선택 후 결제 진입
6. 결제 완료 후 결과 확인 및 재방문

프론트페이지는 이 여정 중 2~4단계를 가장 강하게 지원해야 한다.

메인페이지 권장 섹션 순서는 아래와 같다.

3. 신뢰 포인트 섹션
4. 서비스 상품 섹션
5. 결과 예시 섹션
6. 후기 섹션

10. 푸터

## 4. 섹션별 상세 기획

### 4-1. 헤더

구성 요소

- 로고
- 메뉴: 서비스 소개, 상품, 후기, FAQ
- 우측 CTA 버튼: 지금 사주보기

데스크톱

- 상단 고정 헤더
- 스크롤 전에는 투명 또는 반투명
- 스크롤 시 배경과 블러 추가

모바일

- 좌측 로고
- 우측 햄버거 메뉴
- 하단 고정 CTA 버튼 별도 운영 가능

UX 포인트

- 스크롤 위치에 따라 현재 섹션 메뉴 하이라이트
- CTA는 항상 한 번에 보이게 유지

### 4-2. 히어로 섹션

핵심 메시지 예시

- AI가 읽어주는 당신의 사주 흐름
- 오늘의 운, 관계, 금전, 흐름까지 깊이 있게

구성 요소

- 메인 카피
- 보조 설명
- 주요 CTA: 지금 사주보기
- 보조 CTA: 결과 예시 보기
- 시각 요소: 움직이는 배경, 별자리, 전통 문양, 카드형 결과 미리보기

운영 의도

- 첫 3초 안에 서비스 성격과 감성을 전달
- "AI 사주"가 가볍지 않고 정교한 서비스로 느껴지게 구성

모바일 구성

- 카피를 2줄 이내 중심으로 압축
- 시각 요소는 세로형 카드 1개 중심
- CTA 버튼 2개를 세로로 쌓지 말고 우선순위를 두어 1개 강조

### 4-3. 신뢰 포인트 섹션

목적

- 결제 전 사용자 불안 해소

표현 방식

- 카드 3~4개

추천 항목

- AI 기반 정교한 해석
- 결제 후 즉시 결과 확인
- 연애/재물/직장 등 세부 분석
- 모바일에서도 간편 확인

모바일

- 가로 스와이프 카드 또는 2열 그리드

### 4-4. 서비스 상품 섹션

목적

- 어떤 상품이 있고 가격이 얼마인지 직관적으로 안내

구성 요소

- 서비스 카드 목록
- 인기 배지
- 가격
- 짧은 설명
- CTA 버튼

카드 정보

- 아이콘
- 상품명
- 한 줄 설명
- 가격
- 인기 여부
- 선택 버튼

모바일

- 카드 세로 스택
- 인기 상품은 가장 위
- 가격과 버튼이 첫 화면 안에 들어오게 구성

### 4-5. 결과 예시 섹션

목적

- "결과가 어느 정도 퀄리티인지" 보여주기

구성 요소

- 결과 카드 미리보기
- 연애 / 재물 / 총운 탭
- 일부 모자이크 또는 샘플 데이터

UX 포인트

- 탭 전환 시 내용이 자연스럽게 바뀌는 모션
- 화면 전체가 아닌 카드 내부만 움직여 집중감 유지

모바일

- 탭 + 카드 1개 구성
- 너무 많은 텍스트를 한 번에 보여주지 않음

### 4-6. 후기 섹션

목적

- 실사용자 반응을 통해 신뢰 확보

구성 요소

- 후기 카드 리스트
- 만족도 강조 문구
- 랜딩 노출 후기만 반영

데스크톱

- 2열 또는 3열 카드 배열
- 일부 카드는 높이를 다르게 해 리듬감 부여

모바일

- 스와이프 슬라이더
- 자동 롤링은 느리게

### 4-7. 이용 흐름 섹션

목적

- 서비스 이용 방식을 매우 쉽게 이해시키기

3단계 구성

1. 정보 입력
2. 결제
3. 결과 확인

표현 방식

- 번호형 스텝 카드
- 간단한 아이콘과 설명

모바일

- 세로 타임라인 구조 권장

### 4-8. FAQ 섹션

권장 질문

- 결제 후 바로 결과를 볼 수 있나요
- 모바일에서도 이용 가능한가요
- 환불은 어떻게 되나요
- 개인정보는 어떻게 보호되나요
- 결과는 어디서 다시 볼 수 있나요

UX 포인트

- 아코디언 방식
- 한 번에 하나만 열리게 하여 스크롤 길이 제어

### 4-9. 하단 CTA 섹션

목적

- 마지막 전환 유도

구성 요소

- 강한 한 줄 메시지
- CTA 버튼
- 보조 설명

예시

- 지금 당신의 흐름을 확인해보세요
- 결제 후 바로 결과를 확인할 수 있습니다

## 5. 모바일 우선 설계 원칙

사주명인은 모바일 유입 비중이 높을 가능성이 크므로, 실제 기준은 모바일 우선으로 잡는다.

핵심 원칙

- 첫 화면에서 핵심 카피와 CTA가 바로 보여야 한다.
- 텍스트는 짧고 블록 단위로 끊어 읽기 쉽게 구성한다.
- 섹션 간 여백을 충분히 두어 답답함을 줄인다.
- 버튼은 엄지 터치 영역을 고려해 충분히 크게 만든다.
- 모션은 모바일에서 더 짧고 가볍게 조정한다.

모바일 공통 규칙

- 본문 최대 폭은 좁게 유지
- 카드 간격은 충분히 확보
- 2열보다 1열 우선
- 고정 하단 CTA 적극 활용
- 긴 설명은 아코디언 또는 탭으로 분산

## 7. 섹션별 모션 기획

전체 방향

- 화려함보다 고급스러운 움직임
- 빠른 반짝임보다 느리고 깊은 흐름
- 스크롤에 따라 부드럽게 등장하는 방식 우선

### 7-1. 공통 모션 원칙

- 초기 진입 시 0.4초~0.8초 내의 부드러운 등장
- 큰 요소는 아래에서 위로 12~24px 이동하며 페이드 인
- 작은 배경 요소는 천천히 떠다니는 느낌
- hover는 확대보다 미세한 상승과 그림자 변화 중심

### 7-2. 히어로 모션

- 배경 그라디언트가 아주 천천히 이동
- 별자리 점 또는 입자가 부유
- 메인 카피는 순차 등장
- 결과 카드 mockup은 아주 약한 float motion 적용

### 7-3. 신뢰 포인트 모션

- 카드가 스크롤 진입 시 순차 등장
- 아이콘은 약한 scale-up

### 7-4. 상품 카드 모션

- 카드 hover 시 위로 4~8px 이동
- 인기 상품 카드는 미세한 빛 흐름 효과 가능
- 모바일에서는 hover 대신 터치 시 눌림 반응

### 7-5. 결과 예시 모션

- 탭 전환 시 카드 내용 cross-fade
- 결과 문단은 줄 단위가 아니라 카드 단위로 전환

### 7-6. 후기 모션

- 데스크톱: 카드가 천천히 떠오르는 느낌
- 모바일: 스와이프 중심, 자동 재생은 매우 느리게

### 7-7. FAQ 모션

- 아코디언 열릴 때 높이 전환 + opacity 전환
- 화살표 아이콘 회전

## 8. 성능 및 구현 가이드

- 3D WebGL은 초기 MVP에서 제외 권장
- CSS 애니메이션, Framer Motion, Lottie 정도로 충분
- 모바일에서는 배경 입자 수를 줄인다
- prefers-reduced-motion 대응 필수
- 첫 화면 이미지나 배경 비주얼은 WebP 또는 AVIF 사용

추천 기술 방향

- Next.js
- Tailwind 또는 CSS Modules
- Framer Motion
- Embla Carousel 또는 Swiper

## 9. 디자인 키워드

- 밤하늘, 별자리, 먹빛, 운의 흐름
