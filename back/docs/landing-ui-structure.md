# 사주명인 랜딩페이지 UI 구조안

## 1. 화면 구성 요약

랜딩페이지는 아래 순서로 구성한다.

1. 고정 헤더
2. 히어로
3. 신뢰 포인트
4. 상품 소개
5. 결과 예시
6. 후기
7. 이용 단계
8. FAQ
9. 하단 CTA
10. 푸터

## 2. 섹션별 UI 역할

### 헤더

- 좌측: 로고
- 중앙: 메뉴 링크
- 우측: 주요 CTA
- 모바일: 햄버거 또는 축약 메뉴 + 하단 고정 CTA

### 히어로

- 좌측: 메인 카피, 보조 설명, CTA 2개
- 우측: 결과 카드 또는 비주얼 패널
- 배경: `hero.png`
- 모션: 배경 shimmer, 카드 float

### 신뢰 포인트

- 3~4개 카드
- 핵심 장점 요약
- 카드 진입 애니메이션

### 상품 소개

- 배경: `service.png`
- 카드 리스트
- 인기 상품 강조
- 모바일에서는 1열 스택

### 결과 예시

- 배경 또는 카드 비주얼: `result-card.png`
- 탭형 구조
- 결과 샘플 카피 노출

### 후기

- 배경: `review.png`
- 후기 카드 3개 이상
- 모바일 슬라이드 대응 가능한 구조

### 이용 단계

- 3단계 타임라인
- 입력 > 결제 > 결과 확인

### FAQ

- 아코디언형 질문 목록
- 모바일 읽기 최적화

### CTA

- 배경: `cta.png`
- 짧은 전환 문구 + 큰 CTA

## 3. 모바일 핵심 원칙

- 히어로 카피는 2~3줄 이내
- CTA는 한 번에 한 개를 강하게 보여준다
- 상품 카드는 세로 스택
- 후기와 FAQ는 세로 읽기 흐름 우선
- 이미지 장식보다 텍스트 가독성 우선

## 4. 구현 우선순위

1. 메인 레이아웃
2. 히어로
3. 상품 섹션
4. 후기와 결과 예시
5. FAQ
6. CTA
