🚀 데이터베이스(Supabase) 설정 가이드

중요: 현재 "Table not found" 에러가 발생하는 이유는 Supabase 프로젝트에 필요한 테이블들이 생성되지 않았기 때문입니다. 아래 단계를 따라해 주세요.

1단계: Supabase 접속

Supabase 대시보드에 로그인하고 사용하는 프로젝트를 선택하세요.

2단계: SQL Editor 실행

왼쪽 사이드바에서 SQL Editor 메뉴를 클릭하세요.

3단계: 테이블 생성 SQL 실행

아래 SQL 코드를 복사하여 SQL Editor에 붙여넣고 Run 버튼을 클릭하세요.

-- 1. 서비스 목록 테이블
create table if not exists public.services (
  id uuid primary key default gen_random_uuid(),
  slug text not null unique,
  name text not null,
  display_name text not null,
  description text,
  price integer not null,
  status text not null default 'active',
  is_popular boolean not null default false,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

-- 2. 결제 및 분석 결과 저장 테이블
create table if not exists public.payments (
  id uuid primary key default gen_random_uuid(),
  order_id text not null unique,
  service_id uuid references public.services(id) on delete set null,
  service_name_snapshot text not null,
  buyer_name text,
  buyer_email text,
  buyer_phone text,
  amount integer not null,
  payment_method text not null,
  status text not null,
  portone_payment_id text,
  portone_transaction_id text,
  refund_reason text,
  result_link text,
  result_json jsonb,
  approved_at timestamptz,
  refunded_at timestamptz,
  raw_payload jsonb,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

-- RLS 정책 설정 (간편 설정)
alter table public.services enable row level security;
alter table public.payments enable row level security;
create policy "allow_anon_select" on public.services for select using (true);
create policy "allow_anon_all" on public.payments for all using (true) with check (true);

4단계: 확인

SQL 실행 후 어드민 페이지(admin.html)를 새로고침하여 에러가 사라졌는지 확인하세요.
